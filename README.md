## 🚀 Git 브랜치 전략 (Git Flow)
팀원들은 이 구조에 따라 브랜치를 생성하고 작업합니다.
<br/>
feature/* -> develop -> main
<br/>
- main: 최종 배포용 브랜치 (변경 사항은 반드시 팀 전체 확인 후 머지)
- develop: 기능 개발이 완료되면 머지되는 통합 브랜치 (1명만 확인해도 머지 가능)
- feature/*: 각 기능 개발을 위한 개별 작업 브랜치
<br/><br/><br/><br/>
